# ЦС ПАП

ЦС ПАП — это внутренняя веб-платформа для сопровождения сертификации специалистов. Система объединяет в одном месте регистрацию пользователей, личный кабинет, учет CEU-активностей и часов супервизии, проверку документов, работу с экзаменационными заявками, выдачу сертификатов и публичный реестр.

Этот README описывает не идею проекта, а то, что реально есть в репозитории сейчас.

## Что уже умеет система

С точки зрения пользователя платформа уже покрывает основные рабочие сценарии:

- регистрация и вход по email и паролю
- восстановление пароля
- просмотр и редактирование собственного профиля
- принятие согласия на трансграничную передачу персональных данных
- загрузка файлов в систему
- создание CEU-записей и просмотр их истории
- создание заявок на супервизию и просмотр накопленных часов
- просмотр персонального прогресса по активному циклу сертификации
- подача заявки на проверку документов
- просмотр своих сертификатов
- публичный реестр специалистов и публичные карточки профилей

С точки зрения администрирования в проекте уже реализованы:

- список пользователей с фильтрами и карточкой пользователя
- изменение базовых данных пользователя
- смена роли и целевого уровня сертификации
- ручная корректировка CEU-матрицы пользователя
- ручная корректировка матрицы супервизии пользователя
- просмотр и обработка заявок на проверку документов
- выпуск, редактирование и удаление сертификатов
- просмотр и изменение статусов платежей
- работа с экзаменационными заявками
- экспорт данных по пользователю и выгрузка пользователей в Excel
- создание резервной копии базы данных

## Для кого проект

Платформа рассчитана на несколько типов участников процесса:

- соискатель или специалист, который накапливает CEU, ведет часы супервизии и проходит цикл сертификации
- проверяющий или модератор, который рассматривает записи и заявки
- администратор, который сопровождает пользователей, настраивает прогресс и выпускает сертификаты
- внешний посетитель, который использует публичный реестр без авторизации

В коде сейчас используются роли `STUDENT`, `REVIEWER`, `ADMIN`. Отдельно от ролей существуют квалификационные уровни и группы, которые влияют на требования и прогресс сертификации.

## Как устроен проект

Репозиторий разделен на две основные части:

- `frontend` — клиентское приложение на React
- `backend` — API и бизнес-логика

Дополнительно в корне лежит `docker-compose.yml`, который поднимает PostgreSQL, backend и frontend как отдельные сервисы.

## Основные модули системы

### Аутентификация и профиль

Есть регистрация, логин, получение текущего пользователя, редактирование данных профиля, восстановление пароля и фиксация согласия на трансграничную передачу персональных данных.

### CEU

Пользователь может создавать CEU-записи, видеть текущую сводку и историю. Для администратора есть отдельные инструменты просмотра и ручной правки CEU-матрицы пользователя.

### Супервизия

Реализованы создание заявок на супервизию, история, сводка, режим проверки и редактирования часов, а также отдельная матрица супервизии для администратора.

### Циклы сертификации

В проекте есть понятие активного цикла сертификации или продления. Для цикла хранится целевой уровень, статус, снимок требований и связанные записи CEU и супервизии.

### Проверка документов

Пользователь может подать заявку на проверку документов, а администратор может просматривать список заявок, открывать детали, менять статус и отмечать оплату.

### Сертификаты

Администратор может выпускать сертификаты, обновлять их и удалять. Пользователь может видеть свои сертификаты, а система хранит историю сертификатов, а не только один текущий документ.

### Экзаменационные заявки

Есть отдельный модуль заявок на экзамен: просмотр своей заявки, просмотр общего списка и изменение статуса со стороны администратора.

### Платежи

В системе уже есть сущности платежей и API для просмотра и смены статусов. В схеме данных предусмотрен провайдер оплаты и история webhook-событий.

### Публичный реестр

Без входа в систему доступны список участников реестра и публичная карточка специалиста.

### Уведомления и загрузка файлов

Есть модуль уведомлений, а также отдельные маршруты для загрузки, просмотра, обновления и удаления файлов.

## Что видно по модели данных

Текущая Prisma-схема уже описывает не только пользователей и сертификаты, но и полноценную предметную модель процесса:

- пользователи, роли и группы
- CEU-записи и атомарные CEU-элементы
- записи супервизии и атомарные часы
- циклы сертификации и продления
- сертификаты
- заявки на проверку документов
- экзаменационные заявки
- платежи и webhook-события
- уведомления
- загруженные файлы
- пользовательские согласия
- кастомные города

Это важно, потому что проект уже ближе к рабочей внутренней системе, чем к прототипу или черновому MVP.
